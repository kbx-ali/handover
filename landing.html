How it works

Figma → Plugin → Shareable link.

Three steps. No coding. Your devs get a polished, structured brief.

Design in Figma
Work in your Figma file as usual. Name your section frames clearly with a prefix of // — name the section as it is in the Theme to help the dev.
→
Tag in the plugin
Open the Handover plugin. Set each section to Existing, Modified, or Bespoke and add your notes.
→
Share the link
Hit "Open in Handover." Your devs get a clean, read-only brief — no Figma account needed. You can also export to ClickUp from there.
Setup guide

Up and running
in minutes.

Three steps. No tokens, no hosting, no config. Just install the plugin and sign in.

1
Install the Figma plugin
~2 minutes · One-time setup

Download the plugin zip, then point Figma to the manifest file. You only need to do this once — it stays installed.

Figma Plugin

  1. Unzip figma-plugin.zip — you'll get a folder with manifest.json, code.js, and ui.html inside. Keep it somewhere permanent (like your Documents folder).
  2. Open the Figma desktop app. Click the menu icon (top left) → Plugins → Development → Import plugin from manifest…
  3. Navigate into the unzipped plugin folder and select the manifest.json file.
  4. To run the plugin: right-click anywhere on the Figma canvas → Plugins → Development → Design → Dev Handover.

3
Add your Figma file URL
30 seconds · Final step!

The plugin needs to know which Figma file you're working in so it can generate direct jump links to each section.

  1. In the plugin, click the gear icon (top right of the plugin panel) to open Settings.
  2. Switch to your Figma file's browser tab and copy the full URL from the address bar.
  3. Paste it into the "Figma File URL" field in the plugin settings. Click Save.
  4. Head back to the main view — your frames will be listed. Tag their status, add notes, and hit "Open in Handover" to sync everything to the web app.
